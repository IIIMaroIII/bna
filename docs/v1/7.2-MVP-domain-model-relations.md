### ================================= MVP Domain Model Relations =================================

### 1 -> N (one to many)
    - Company `1 -> N` User
        `User.company_id == Company.id`
    - Company `1 -> N` Team
        `Team.company_id == Company.id`
    - Company `1 -> N` Project
        `Project.company_id == Company.id`
    - Company `1 -> N` WorkSession
        `WorkSession.company_id == Company.id`
    - Company `1 -> N` AllowedCompanyNetwork
        `AllowedCompanyNetwork.company_id == Company.id`
    - PricingPlan `1 -> N` PricingPlanBenefit
        `PricingPlanBenefit.pricing_plan_id == PricingPlan.id`
### N -> N (many to many)
    - User `1 -> N` TeamMember
        `TeamMember.user_id == User.id`
    - Team `1 -> N` TeamMember
        `TeamMember.team_id == Team.id`
    *** User `N -> N` Team --> TeamMember(user_id,team_id)
    - Team `1 -> N` ProjectTeamAssignment
        `ProjectTeamAssignment.team_id == Team.id`
    - Project `1 -> N` ProjectTeamAssignment
        `ProjectTeamAssignment.project_id == Project.id`
    *** Team `N -> N` Project через ProjectTeamAssignment(team_id, project_id)
    - User `1 -> N` WorkSession
        `WorkSession.user_id == User.id`
    - Team `1 -> N` WorkSession
        `WorkSession.team_id == Team.id`
    - Project `1 -> N` WorkSession
        `WorkSession.project_id == Project.id`
    - Editor link (кто правил admin/owner)
        `WorkSession.updated_by_user_id == User.id`

### Standalone (Public) (без обязательных связей)
    - FeedbackMessage
    - Feature
    - PricingPlan
