# Порядок работы над продуктом

1. **Контекст**  
   Определить тип продукта (B2B/B2C, SaaS), общую цель и рамки проекта.

2. **Пользователь (WHO)**  
   Зафиксировать, кто пользуется системой, какие есть роли и их ответственность.

3. **Боль (WHY)**  
   Определить конкретную проблему, которую продукт решает прямо сейчас.

4. **Ожидаемый результат (OUTCOME)**  
   Описать, что изменится для пользователя после внедрения продукта.

5. **MVP (SCOPE)**  
   Определить минимальный набор функций, достаточный для решения боли.

6. **Пользовательские сценарии (USER FLOWS)**  
   Описать пошаговые действия пользователей для достижения результата.

7. **Экраны (UI MAP)**  
   Сформировать список экранов, необходимых для реализации сценариев.

8. **Functional Requirements (FR)**  
   Зафиксировать, что система должна уметь делать.

9. **Non-Functional Requirements (NFR)**  
   Определить требования к качеству, надёжности и безопасности системы.

10. **Архитектура**  
    Спроектировать техническую реализацию (модули, сервисы, БД).

11. **Декомпозиция в задачи**  
    Разбить требования на user stories и задачи для команды.
